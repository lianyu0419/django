1. Introductions

comment comment is a commenting system building on top of django.

2. Installation

You should to copy the following apps into your project:
  • comment
  • acount

    application acount is responsible for login and logout

  • html5tags

    html5tags provide basic style for comment form and pagination

  • toollib

    toollib is a common module for rendering template, pagination etc

  • django_avatar

    avatar should be installed for handling user avatars. invoke the following command:

    easy_install django_avatar

3. Configuration

  1. You should add the following to your settings.py

    INSTALLED_APPS += ('comment','acount','html5tags','toollib', 'avatar',)

  2. You should add the following to your urls.py

    urlpatterns += patterns('',
                          url(r'', include('comment.urls')),
                          url(r'auth/', include('account.urls')),
                          )

4. Instructions

  1. you should use syncdb to install comment, account and avatar model before running.

    python manage.py syncdb

  2. Wherever you want to display comments for one object, first load the comments template tags:

    {% load comment_tags %}

  3. Then, use the comment_list tag to display comments:

    {% comments app_name model_name object_pk prefix pageno request  %}

    object_pk: id of some object(e.g. article or blog entry) which is commented

    model_name: the object's model name

    app: the app which the object belongs to

5. Global Settings

There are a number of settings available to easily customize the comments that appear on the site. You can find them in file config.py. Listed below are
  1. PAGE_SIZE = 10

    PAGE_SIZE is an integers displaying amount of comments each page. Defaults to 10
